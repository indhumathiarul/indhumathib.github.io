What is gRPC? Beginner — Core Concept

💡
Simple Answer gRPC lets one computer call a function on another computer over the network — as if that function lived locally. It's like REST API but faster, strongly typed, and built for microservices.

The Analogy 🍕

Imagine ordering pizza on a phone call vs texting. REST API is like texting — flexible but wordy (JSON). gRPC is the phone call — direct, efficient, structured. The menu is your .proto file. The order items are your messages. The call itself is the RPC.

Architecture Overview

┌─────────────────────────────────────────────────────┐ │ CLIENT APP │ │ ┌──────────────────────────────────────────────┐ │ │ │ Generated Stub (C# client) │ │ │ │ grpcClient.SayHello(request) ◄──── you call│ │ │ └──────────────┬───────────────────────────────┘ │ └─────────────────┼───────────────────────────────────┘ │ HTTP/2 (binary, multiplexed) │ Protobuf serialized bytes │ ┌─────────────────┼───────────────────────────────────┐ │ ▼ SERVER APP │ │ ┌──────────────────────────────────────────────┐ │ │ │ gRPC Server (ASP.NET Core) │ │ │ │ GreeterService.SayHello(context) ◄─ runs │ │ │ └──────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────┘

✅ gRPC Strengths

  • Strongly typed contracts (.proto)
  • ~7x faster than REST/JSON
  • Supports streaming (real-time)
  • Code generated automatically
  • Works in 10+ languages
  • Built-in deadlines & cancellation

⚠️ Limitations

  • Not human readable (binary)
  • Browser support needs gRPC-Web
  • Steeper learning curve
  • Proto schema must be shared
  • Harder to debug raw traffic

Protocol Buffers (.proto) Beginner — The Contract

📋
The .proto file IS your API contract It defines services (what functions exist), messages (data shapes), and field types. All code is auto-generated from it — never write serialization code manually.

Anatomy of a .proto File

PROTOBUF
// syntax version — always use proto3
syntax = "proto3";

// namespace in generated C# code
option csharp_namespace = "MyApp.Grpc";

// logical grouping
package greeter;

// ── SERVICE: defines the RPC methods ──────────────
service Greeter {
  // Unary: 1 request → 1 response
  rpc SayHello (HelloRequest) returns (HelloReply);

  // Server streaming: 1 request → many responses
  rpc SayHellos (HelloRequest) returns (stream HelloReply);
}

// ── MESSAGES: define data structures ──────────────
message HelloRequest {
  string  name = 1;   // field number (NOT value!)
  int32   age  = 2;
}

message HelloReply {
  string  message    = 1;
  bool    is_success = 2;
}

Scalar Types

Proto TypeC# Type
stringstring
boolbool
int32int
int64long
floatfloat
doubledouble
bytesByteString

Complex Types

// Repeated (list)
message OrderList {
  repeated string items = 1;
}

// Map (dictionary)
message Config {
  map<string, string> settings = 1;
}

// Enum
enum Status {
  UNKNOWN  = 0;  // must start at 0!
  ACTIVE   = 1;
  INACTIVE = 2;
}

// Nested message
message User {
  Address address = 1;  // another message
}
⚠️
Field Numbers Are Forever! Once you assign a field number (e.g., name = 1), never reuse it even if you delete that field. Use reserved 5; to mark retired numbers. Changing them breaks backward compatibility.

Unary RPC in C# Basic — The Most Common Pattern

📋
Unary RPC = one request, one response. Exactly like a regular method call, just over the network. This covers 80% of gRPC use cases.

Server-Side Implementation

Services/GreeterService.cs
using Grpc.Core;
using MyApp.Grpc;

public class GreeterService : Greeter.GreeterBase
{
    private readonly ILogger<GreeterService> _logger;

    public GreeterService(ILogger<GreeterService> logger)
        => _logger = logger;

    // Override the generated base method
    public override async Task<HelloReply> SayHello(
        HelloRequest request,
        ServerCallContext context)    // deadline, metadata, cancellation
    {
        _logger.LogInformation("Received: {Name}", request.Name);

        // Business logic here
        await Task.Delay(10);  // simulate async work

        return new HelloReply
        {
            Message   = $"Hello, {request.Name}!",
            IsSuccess = true
        };
    }
}

Client-Side Call

Program.cs (Client)
using Grpc.Net.Client;
using MyApp.Grpc;

// 1. Create the channel (connection to server)
using var channel = GrpcChannel.ForAddress("https://localhost:7042");

// 2. Create the typed client stub
var client = new Greeter.GreeterClient(channel);

// 3. Make the RPC call — looks like a normal method!
var reply = await client.SayHelloAsync(new HelloRequest
{
    Name = "Indhu",
    Age  = 25
});

Console.WriteLine($"Server replied: {reply.Message}");
Console.WriteLine($"Success: {reply.IsSuccess}");

Using gRPC with Dependency Injection (.NET Client)

ServerCallContext — What's Available

public override async Task<HelloReply> SayHello(
    HelloRequest request, ServerCallContext context)
{
    // Deadline set by client
    var deadline    = context.Deadline;

    // Cancellation token — respect this!
    var token       = context.CancellationToken;

    // Headers sent by client (like HTTP headers)
    var authHeader  = context.RequestHeaders.GetValue("authorization");

    // Peer address
    var peer        = context.Peer;

    // Write response trailers (metadata sent after response)
    context.ResponseTrailers.Add("x-request-id", Guid.NewGuid().ToString());

    return new HelloReply { Message = "Done" };
}

Streaming RPC Intermediate — Real-Time Data

The 4 RPC Patterns

1️⃣ Unary
Client ──[req]──► Server Client ◄─[res]── Server
One request, one response
2️⃣ Server Streaming
Client ──[req]──► Server Client ◄─[r1]── Server Client ◄─[r2]── Server Client ◄─[r3]── Server
One request, many responses
3️⃣ Client Streaming
Client ──[m1]──► Server Client ──[m2]──► Server Client ──[m3]──► Server Client ◄─[res]── Server
Many requests, one response
4️⃣ Bidirectional
Client ──[m1]──► Server Client ◄─[r1]── Server Client ──[m2]──► Server Client ◄─[r2]── Server
Many requests, many responses

Server Streaming — Live Price Feed

price_feed.proto
rpc StreamPrices (SubscribeRequest) returns (stream PriceUpdate);
Server
public override async Task StreamPrices(
    SubscribeRequest request,
    IServerStreamWriter<PriceUpdate> responseStream,
    ServerCallContext context)
{
    while (!context.CancellationToken.IsCancellationRequested)
    {
        var price = await _priceService.GetLatestAsync(request.Symbol);

        // Write each item to the stream
        await responseStream.WriteAsync(new PriceUpdate
        {
            Symbol    = request.Symbol,
            Price     = price,
            Timestamp = Timestamp.FromDateTime(DateTime.UtcNow)
        });

        await Task.Delay(1000, context.CancellationToken);
    }
}
Client
var call = client.StreamPrices(new SubscribeRequest { Symbol = "AAPL" });

// Read from stream using async enumerable
await foreach (var update in call.ResponseStream.ReadAllAsync())
{
    Console.WriteLine($"{update.Symbol}: ${update.Price}");
}

Client Streaming — Upload Chunks

file_upload.proto
rpc UploadFile (stream FileChunk) returns (UploadResult);
Client
using var call = client.UploadFile();

// Stream file in 64KB chunks
var buffer = new byte[65536];
int bytesRead;

using var fileStream = File.OpenRead("largefile.zip");
while ((bytesRead = await fileStream.ReadAsync(buffer)) > 0)
{
    await call.RequestStream.WriteAsync(new FileChunk
    {
        Content = ByteString.CopyFrom(buffer, 0, bytesRead)
    });
}

// Signal done — mandatory!
await call.RequestStream.CompleteAsync();

var result = await call;
Console.WriteLine($"Uploaded {result.BytesReceived} bytes");

Bidirectional Streaming — Live Chat

chat.proto
rpc Chat (stream ChatMessage) returns (stream ChatMessage);
Client — Send and Receive Concurrently
using var call = client.Chat();

// Read task — runs in background
var readTask = Task.Run(async () =>
{
    await foreach (var msg in call.ResponseStream.ReadAllAsync())
        Console.WriteLine($"[{msg.From}]: {msg.Text}");
});

// Write messages from console
string? line;
while ((line = Console.ReadLine()) != null)
{
    await call.RequestStream.WriteAsync(new ChatMessage
    {
        From = "Indhu",
        Text = line
    });
}

await call.RequestStream.CompleteAsync();
await readTask;

[Authorize]                   // whole service requires auth
public class GreeterService : Greeter.GreeterBase
{
    [AllowAnonymous]           // override for specific method
    public override Task<HelloReply> SayHello(...) { ... }

    [Authorize(Roles = "Admin")]  // role-based
    public override Task<AdminReply> AdminAction(...) { ... }

        SecureRequest req, ServerCallContext context)
    {
        // Get the authenticated user

        return Task.FromResult(new SecureReply { UserId = name });
    }
}

Interceptors (Middleware) Advanced — Cross-Cutting Concerns

🧩
Interceptors = gRPC Middleware Intercept every RPC call for logging, auth validation, retry logic, metrics, or correlation IDs — without touching service code.

Server Interceptor — Logging + Timing

LoggingInterceptor.cs
using Grpc.Core;
using Grpc.Core.Interceptors;

public class LoggingInterceptor : Interceptor
{
    private readonly ILogger<LoggingInterceptor> _logger;

    public LoggingInterceptor(ILogger<LoggingInterceptor> logger)
        => _logger = logger;

    // Intercept unary calls
    public override async Task<TResponse> UnaryServerHandler<TRequest, TResponse>(
        TRequest request,
        ServerCallContext context,
        UnaryServerMethod<TRequest, TResponse> continuation)
        where TRequest : class
        where TResponse : class
    {
        var sw = Stopwatch.StartNew();
        _logger.LogInformation("➡ {Method}", context.Method);

        try
        {
            var response = await continuation(request, context);
            _logger.LogInformation("✅ {Method} in {Ms}ms",
                context.Method, sw.ElapsedMilliseconds);
            return response;
        }
        catch (Exception ex)
        {
            _logger.LogError(ex, "❌ {Method} failed", context.Method);
            throw;
        }
    }
}
Program.cs — Register Interceptor
builder.Services.AddGrpc(options =>
{
    options.Interceptors.Add<LoggingInterceptor>();
});

// Or per-service:
app.MapGrpcService<GreeterService>()
   .AddInterceptor<LoggingInterceptor>();

Client Interceptor — Add Auth Header

AuthInterceptor.cs
public class AuthInterceptor : Interceptor
{
    private readonly ITokenProvider _tokens;

    public AuthInterceptor(ITokenProvider tokens) => _tokens = tokens;

    public override AsyncUnaryCall<TResponse>
        AsyncUnaryCall<TRequest, TResponse>(
            TRequest request,
            ClientInterceptorContext<TRequest, TResponse> context,
            AsyncUnaryCallContinuation<TRequest, TResponse> continuation)
        where TRequest : class
        where TResponse : class
    {
        var headers = new Metadata();
        headers.Add("authorization", $"Bearer {_tokens.GetToken()}");

        var newContext = new ClientInterceptorContext<TRequest, TResponse>(
            context.Method, context.Host,
            context.Options.WithHeaders(headers));

        return continuation(request, newContext);
    }
}

// Register on client channel:
var invoker = channel.Intercept(new AuthInterceptor(tokenProvider));
var client  = new Greeter.GreeterClient(invoker);

Load Balancing Advanced — High Availability

⚠️
L7 vs L4 Load Balancing gRPC runs on HTTP/2 with long-lived connections. Traditional L4 load balancers (TCP) can't distribute individual RPCs — they only balance new connections. Use L7 (application-layer) load balancing like Envoy, Nginx, or .NET's built-in client-side balancing.

Client-Side Load Balancing

Program.cs
// Round-robin across multiple server addresses
var channel = GrpcChannel.ForAddress("static:///",
    new GrpcChannelOptions
    {
        Credentials = ChannelCredentials.Insecure,
        ServiceConfig = new ServiceConfig
        {
            LoadBalancingConfigs = {
                new RoundRobinConfig()
            }
        }
    });

// Register multiple endpoints
var addresses = new[] {
    "https://server1:7042",
    "https://server2:7042",
    "https://server3:7042"
};

builder.Services
    .AddGrpcClient<Greeter.GreeterClient>()
    .ConfigureChannel(options =>
    {
        options.ServiceConfig = new ServiceConfig
        {
            LoadBalancingConfigs = { new RoundRobinConfig() }
        };
    });

Throwing Errors on Server

public override Task<OrderResponse> GetOrder(
    GetOrderRequest request, ServerCallContext context)
{
    if (string.IsNullOrEmpty(request.OrderId))
    {
        // Throw RpcException with proper status code
        throw new RpcException(new Status(
            StatusCode.InvalidArgument,
            "OrderId cannot be empty"));
    }

    var order = _db.FindOrder(request.OrderId);
    if (order == null)
    {
        throw new RpcException(new Status(
            StatusCode.NotFound,
            $"Order {request.OrderId} not found"));
    }

    return Task.FromResult(MapToResponse(order));
}

Handling Errors on Client

try
{
    var reply = await client.GetOrderAsync(request,
        deadline: DateTime.UtcNow.AddSeconds(5));
}
catch (RpcException ex) when (ex.StatusCode == StatusCode.NotFound)
{
    Console.WriteLine("Order does not exist!");
}
catch (RpcException ex) when (ex.StatusCode == StatusCode.DeadlineExceeded)
{
    Console.WriteLine("Server too slow — timed out!");
}
catch (RpcException ex) when (ex.StatusCode == StatusCode.Unavailable)
{
    Console.WriteLine("Server down — retry later");
}
catch (RpcException ex)
{
    Console.WriteLine($"gRPC Error: {ex.Status.Detail}");
}

gRPC-Web & Transcoding Advanced — Browser Support

🌐
Browsers can't use gRPC directly Browsers don't support HTTP/2 trailers required by gRPC. gRPC-Web is a proxy layer that works around this limitation. For full REST compatibility, use HTTP API transcoding.

Setup gRPC-Web in ASP.NET Core

dotnet add package Grpc.AspNetCore.Web
Program.cs
builder.Services.AddGrpc();

var app = builder.Build();

// Must be before UseRouting/MapGrpcService
app.UseGrpcWeb();

app.MapGrpcService<GreeterService>()
   .EnableGrpcWeb();   // ← browser accessible!

app.Run();

HTTP API Transcoding — REST + gRPC at Once

greet.proto
import "google/api/annotations.proto";

service Greeter {
  rpc SayHello (HelloRequest) returns (HelloReply) {
    option (google.api.http) = {
      post: "/v1/hello"          // REST endpoint
      body: "*"
    };
  }

  rpc GetUser (GetUserRequest) returns (User) {
    option (google.api.http) = {
      get: "/v1/users/{user_id}"  // path parameter
    };
  }
}

Message Size & Performance Tuning

builder.Services.AddGrpc(options =>
{
    // Max receive message size (default 4MB)
    options.MaxReceiveMessageSize = 16 * 1024 * 1024;  // 16MB

    // Max send message size (default unlimited)
    options.MaxSendMessageSize    = 16 * 1024 * 1024;

    // Enable response compression
    options.ResponseCompressionLevel    = CompressionLevel.Optimal;
    options.ResponseCompressionAlgorithm = "gzip";
});

Production Checklist

  • TLS enabled on all channels (never plaintext in prod)
  • Deadlines set on every client call (prevents hangs)
  • Retry policy configured for transient errors (UNAVAILABLE)
  • Health check endpoint exposed for load balancers
  • OpenTelemetry tracing + metrics enabled
  • Channel reused (singleton), not created per-request
  • Message size limits configured explicitly
  • Response compression enabled for large payloads
  • Interceptors for logging, auth, and correlation IDs
  • Rate limiting applied at gateway level
  • Proto field numbers never reused after deletion
  • Reflection disabled in production

Schema Evolution — Backward Compatibility

✅ Safe Changes

  • Add new fields (any type)
  • Add new RPC methods
  • Add new enum values
  • Change field name (number stays)

❌ Breaking Changes

  • Change field numbers
  • Change field types
  • Remove required fields
  • Change RPC signatures

Common Mistakes & Fixes

MistakeFix
No deadline setAlways pass deadline: DateTime.UtcNow.AddSeconds(N)
Channel per requestRegister channel/client as singleton
Reusing field numbersUse reserved for deleted fields
HTTP/1.1 for gRPCSet Protocols = Http2 in Kestrel
No cancellation handlingRespect context.CancellationToken in loops
Forget CompleteAsync()Always call on client streams when done
